# require-cl-cffi

A library for racket that allows you to import Common Lisp C FFI bindings, such as those generated by [cl-bindgen](https://github.com/sdilts/cl-bindgen). This assumes that the Common Lisp file being imported only defines C FFI bindings, and does not contain any other Common Lisp syntax or identifiers that are not defined in Racket.

## How?

Common Lisp and Racket have close enough syntax that the default Racket reader can actually parse a lot of Common Lisp without any errors. Therefore, defining macros that mimic the Common Lisp C FFI system to transform these calls to their Racket equivalents would work just fine.

## Why??

Racket has a number of methods of interfacing with C libraries, and a number of bindings already out there as packages. However, creating new bindings can be tedious, and largely automatable. Since such an automation already exists for such a similar language, it would be easier to translate the bindings generated by cl-bindgen than create a Racket equivalent from scratch.
